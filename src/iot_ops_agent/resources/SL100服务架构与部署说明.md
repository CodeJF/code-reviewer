# 匿名 IoT 服务架构与排障说明

本文是公开仓库使用的匿名参考资料，不包含真实主机、索引、账号或生产路径。

## 服务职责

- `gateway`：接收设备 HTTP 登录、鉴权和接入请求，并把设备连接信息交给 `deviceShadow`。设备无法登录时先检查 gateway 的 access 与 error 日志。
- `deviceShadow`：维护设备在线状态、MQTT 会话和 App WebSocket 连接。设备反复上下线、MQTT 断开、WebSocket 握手失败都应检查该服务。
- `pushService`：消费 Redis 队列并向 App 推送告警、状态变化和离线消息。推送缺失时同时检查 Redis backlog 与 pushService error 日志。
- `AdminService`：提供管理端业务接口，不直接维护设备长连接。
- `cloudStorage`：处理对象存储和归档任务。

## 推荐排障路径

设备 MQTT 连不上时，先从 gateway 登录请求确认身份与接入参数，再检查 deviceShadow 的 MQTT connect、disconnect 和鉴权证据。如果设备已在线但 App 没有消息，再检查 Redis 队列与 pushService。

所有查询都应限定服务与时间范围。报告只保留脱敏 evidence，并由人工决定是否升级 Incident。
